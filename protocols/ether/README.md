# ether

The `ether` protocol carries public, connectionless messages across a broadcast network such as a local area network. It is used for node presence and discovery: a node pushes a small typed object to every peer reachable on the same broadcast segment, and listening nodes learn who is nearby.

Messages travel over UDP. The node binds a single UDP socket on the ether port (8822 by default) and sends each message as one datagram to the IPv4 broadcast address of every eligible, non-loopback, non-link-local interface, deduplicating addresses shared across interfaces. A message may also be unicast to a single IP. Every datagram carries one `mod.ether.signed_broadcast`: a `mod.ether.broadcast` payload — timestamp, source identity, and the object being announced — together with a signature. The signature is produced by the source's node key over the object ID of the broadcast payload, so it covers the payload but not the signature field itself. On receipt a node drops its own broadcasts, drops any datagram without a signature, and verifies the signature against the source identity before delivering the object; each accepted broadcast is surfaced locally as a `mod.ether.events.broadcast_received` event.
